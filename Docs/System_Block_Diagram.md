# System Block Diagram

**Project Name:** STM32 Project 02 – ADC-Based Weighing Machine

**Version:** 1.0

---

# 1. Purpose

This document describes the hardware-level architecture of the ADC-Based Weighing Machine.

The block diagram illustrates how the different hardware components interact with the STM32F407 microcontroller.

---

> **Note:** During development, the STM32 internal temperature sensor is used only to validate the ADC driver. In a real weighing machine, this input would be replaced with an external analog sensor or load-cell interface.

# 3. Hardware Components

| Component | Description |
|-----------|-------------|
| STM32F407VG | Main Microcontroller |
| ADC1 | Converts analog voltage to digital value |
| Internal Temperature Sensor | Used for ADC validation |
| ST-Link | Programming and Debugging |
| User Button | Reserved for future enhancements |

---

# 4. Signal Flow

1. Analog voltage is generated by the sensor.
2. ADC1 samples the analog signal.
3. The ADC converts the signal into a 12-bit digital value.
4. The CPU reads the converted value.
5. The firmware converts the ADC value into a voltage.
6. The application calculates the corresponding weight.
7. The processed result becomes available to the application layer.

---

# 5. Future Hardware Expansion

The architecture supports future integration of:

- External potentiometer
- Load cell
- External ADC (e.g., 24-bit)
- LCD/OLED display
- UART debugging interface
